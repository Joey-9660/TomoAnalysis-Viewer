1. Overview

What is the PDD Framework?

The Phenotype-to-Decision (PDD) Mapping Framework is a systematic methodology for connecting label-free cellular phenotype observations from Tomocube holotomography systems to actionable decisions in drug discovery, drug safety, and translational research.

Unlike traditional endpoint assays that measure single parameters, the PDD framework leverages the rich, multi-parametric data from 3D refractive index (RI) tomograms to build a comprehensive phenotypic profile of drug-treated cells. This profile is then mapped against known drug-response patterns to support decision-making.

Framework Components

Drug-Phenotype Mapping Table

Links drug classes and compounds to expected cellular phenotype changes observable via holotomography. 8 drug classes with 23+ phenotype associations.

Phenotype Descriptor Library

Core Metric Library

2. PDD Workflow

The PDD framework follows a 5-step workflow from data acquisition to decision support:

1

Data Acquisition

Acquire 3D holotomography (HT) and optional fluorescence (FL) data using Tomocube systems (HT-X1, HT-2H). Set up time-lapse protocols for kinetic analysis.

Output: 3D RI tomogram datasets (.h5, .tiff)

2

Phenotype Identification

Use TomoAnalysis Viewer to visualize data and identify cellular phenotypes from the Phenotype Descriptor Library. Compare treatment groups to controls using 2D/3D views and orthogonal slicing.

Reference: Phenotype Descriptors (PHE-001 through PHE-023)

3

Metric Quantification

Quantify observed phenotypes using the Core Metric Library. Use the Viewer's built-in tools (Histogram, Intensity Plot, Line Measurement, Area Plot) mapped to each metric.

Reference: Core Metrics (MET-001 through MET-022)

4

Drug-Phenotype Mapping

Map the quantified phenotype profile to the Drug-Phenotype Mapping Table. Match observed patterns to known drug-class responses. Identify concordant and discordant phenotypes.

Reference: Drug-Phenotype Mappings (DPM-001 through DPM-008)

5

Decision Support

Generate actionable insights based on the mapping results. Apply decision criteria (efficacy indicators, resistance indicators, toxicity flags) to support go/no-go decisions.

Output: Phenotypic Drug Response Report

6. Use Cases

Use Case 1: Oncology Drug Screening

Evaluate compound efficacy across a panel of cancer cell lines using phenotypic profiling.

Workflow: Treat cells with test compound at multiple concentrations → Acquire HT data at 24h, 48h, 72h → Quantify MET-001 (Volume), MET-003 (RI), MET-008 (Dry Mass), MET-017 (Death Rate) → Map to DPM-001 through DPM-006 patterns → Classify mechanism of action → Determine IC50 and therapeutic index.

Use Case 2: Immunotherapy Potency Assessment

Assess checkpoint inhibitor or CAR-T efficacy in immune cell-tumor co-culture systems.

Workflow: Set up immune-tumor co-culture → Add checkpoint inhibitor → Time-lapse HT acquisition over 5 days → Quantify MET-015 (Clustering), MET-016 (Contact Ratio), MET-017 (Death Rate) → Map to DPM-004 patterns → Assess immune activation and tumor killing.

Use Case 3: Hepatotoxicity Safety Screening

Evaluate drug candidates for liver toxicity using primary hepatocytes or iPSC-derived hepatocytes.

Workflow: Treat hepatocytes with test compound → Acquire HT data at 24h, 48h, 72h → Quantify MET-021 (Lipid Droplets), MET-001 (Volume), MET-003 (RI), MET-022 (RI Entropy) → Map to DPM-008 patterns → Flag hepatotoxicity risk → Report DILI potential.

Use Case 4: ADC Selectivity Profiling

Evaluate antibody-drug conjugate selectivity between target-positive and target-negative cell lines.

Workflow: Treat Ag+ and Ag- cell lines with ADC → Acquire HT data → Quantify MET-018 (Selectivity), MET-019 (Internalization Rate), MET-017 (Death Rate) → Map to DPM-006 patterns → Determine therapeutic window and bystander effect.
